# Цифровой Орион

Проект предназначен для подготовки собранных во время полётов данных к последующему обучению моделей маршрутизации и принятия решений.

## Цели и требования

- Сбор данных полёта
- Хранение и обработка в удобном формате
- Python, наличие подробной документации

## Этапы работ

1. Реализация компонентов инфраструктуры.
2. Разработка и интеграция модулей, отвечающих за сбор, обработку и хранение данных.
3. Тестирование и отладка: проверка работоспособности всех компонентов, выявление и устранение ошибок.
4. Проведение оценки производительности.

## Ожидаемый результат

Запускается виртуальная/контейнерная среда, в которой работает программа для сбора, обработки и хранения данных полётов. Эти данные затем используются для обучения системы, чтобы она могла лучше выбирать маршруты.

## Технологический стек (по коду репозитория)

- Python 3.11 (скрипты, обработка данных)
- PX4 SITL, QGroundControl (планирование, сбор ULog)
- Gazebo Classic/Garden (SITL), jMAVSim (альтернатива)
- Docker + Docker Compose (повторяемая среда)
- PX4‑Autopilot (подмодули/зависимости)
- Форматы: ULog → CSV/Parquet
- Документация: Sphinx (Furo, MyST), PlantUML; публикация через GitHub Actions
